## Ближайшие приоритеты (Фаза 5)
1. **Наполнение датасетов** — подключить реальные FAQ/bugs/changelog, включить эмбеддинги (`/api/datasets/sync`).
2. **Повышение точности RAG** — расширить метаданные, настроить rerank, прогнать тесты (`tests/test_staffprobot_queries.json`).
3. **Мониторинг качества** — автоматический отчёт по результатам тестов (accuracy, keyword score, источники).

## Поддержка автообновления (Фаза 6)
- Настроить вебхуки в GitHub для `staffprobot` и `project-brain` (см. `GITHUB_AUTO_UPDATE_GUIDE.md`).
- Добавить уведомления о завершении индексации (опционально Telegram/Slack).
- Зафиксировать процедурный check-list для deploy (CI/CD pipeline уже содержит анализ архитектуры и sync датасетов).

## Среднесрочные задачи
- **Telegram/веб-виджет поддержки** (интеграция RAG в пользовательские каналы).
- **Context rules для Cursor** (автоматический экспорт релевантных правил).
- **Отчётность** — регулярная выгрузка метрик (`/api/stats`, `/api/webhook/status`).

## Отложенные идеи
- Экспорт графа архитектуры в Neo4j/визуализатор.
- Интеграция с DevOps Dashboard StaffProBot (подписка на уведомления Brain).
- Few-shot обучающие примеры для LLM и кэширование популярных вопросов.
